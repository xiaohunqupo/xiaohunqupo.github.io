# 前言
在添加TFS备份任务时，我遇到了如下几个问题：

1. TF401009:Azure DevOps 服务账户 NT AUTHORITY\LOCAL SERVICE 无法访问网络共享。请更改为可访问备份路径的账户。
2. SQL实例“XXXXX”上的数据库“XXXXXX”正在使用简单恢复模式，该模式不能用于拆功能键所有 Azure DevOps 数据库的事务一致性备份。所有Azure DevOps 数据库都必须使用完整恢复模式来创建备份计划。
3. TF401002: SQL Server 数据库引擎未能将数据库备份保存到路径 \\xxxxx\source_backup。  请向 SQL 服务帐户授予对该文件夹的读/写访问权限。

# TF401009
该问题可以通过切换 `Azure DevOps Server服务账户` 解决。
1. 启动 `Azure DevOps Server 管理控制台` 。
2. 在 `服务器名称-应用层`页面的右侧，点击`更改账户`
   ![更改账户](\images\2026-03-11\5.png)
3. 在`更改服务账户`处，选择`使用用户账户`，并输入有备份路径权限的客户即可。注意这个用户全新建议较高，避免影响当前服务运行。
   ![使用用户账户](\images\2026-03-11\6.png)

# 完整恢复模式
该问题是由于错误信息中的数据库可能是经过恢复、还原或人工调整过数据库的备份模式导致的。
正常TFS默认创建的数据库都是完整恢复模式。
修改数据库模式方法如下：
1. 启动 `Microsoft SQL Server Management Statudio` (`SSMS`)。
2. 连接 `SQL Server 服务器`
3. 在数据库中找到前面错误提示信息中的数据库。
4. 点击`右键菜单-属性`
   ![数据库属性](\images\2026-03-11\1.png)
5. 在属性对话框中，将 `选项-恢复模式` 中修改为 `完整`。
   ![恢复模式](\images\2026-03-11\2.png)
6. 修改后由，重启 `SQL Server 服务器`

# TF401002
该错误原因是因为 `SQL Server` 服务没有对备份路径的访问权限。

1. 执行 `services.msc` 启动服务管理程序。
2. 找到 `SQL Server` 服务，点击 `右键菜单-属性`。
   ![服务属性](\images\2026-03-11\3.png)
3. 选择 `登录-此账户` ，并输入有权限访问备份路径的 `本机账户`。需要注意的是，这里的账户如果权限不够又可能导致其它运行问题，所以我直接使用了管理员账户。
   ![登录](\images\2026-03-11\4.png)
4. 重启 `SQL Server` 服务。
